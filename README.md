## Project Description
This project presents an automated impact crater detection system for Mars using a semi-supervised, auto-iterative self-training approach. A YOLOv5 object detection model is trained on a small set of manually labelled craters and iteratively expanded using high-confidence pseudo-labels generated by an IoU-based ensemble module. The system detects unlabelled small craters (< 1 km diameter) that are absent from the initial training catalogue, significantly reducing the manual labelling burden compared to fully supervised methods.

The approach is described in detail in:

> Zhao, J., Davis, J., & Collins, G. S. (2024). *Automated impact crater detection using a self-training machine learning method*. Manuscript submitted to *Earth and Space Science*.

The crater detection software is archived on Zenodo: [doi:10.5281/zenodo.13970867](https://doi.org/10.5281/zenodo.13970867)

The source code for YOLOv5 is available at the [official YOLOv5 repository](https://github.com/ultralytics/yolov5).

## Data Source
The training dataset is based on the 2020 version of the Robbins and Hynek Mars Crater Database (MCD2020), which catalogs over 392,431 craters larger than 1 km in diameter, combined with THEMIS Daytime IR mosaic images.

- The Robbins and Hynek Crater Database: [http://craters.sjrdesign.net/](http://craters.sjrdesign.net/)
- THEMIS images: [USGS Astrogeology Science Center](https://astrogeology.usgs.gov/search?target=&system=&p=1&accscope=&searchBar=)
- The Mars crater detection catalog generated by this work: [doi:10.5281/zenodo.13983674](https://doi.org/10.5281/zenodo.13983674)

## Getting Started

### Prerequisites
- Python 3.8 or later
- A CUDA-capable GPU (8 GB+ VRAM recommended for training)
- A Google account with Google Drive (for running the Colab notebook)

### Installation

Clone this repository:

```bash
git clone https://github.com/ACSE-jz1121/Crater_detection_self_training.git
cd Crater_detection_self_training
```

Install Python dependencies:

```bash
pip install -r requirements.txt
```

Clone the YOLOv5 repository into your working directory (required for training and detection):

```bash
git clone https://github.com/ultralytics/yolov5
```

### Google Colab Setup

1. Upload the cloned repository folder to your Google Drive.
2. Open `Auto_iterative_self_training.ipynb` directly in Google Colab.
3. Mount your Google Drive when prompted and verify the base paths in the notebook match your Drive layout.
4. Replace `YOUR_ROBOFLOW_API_KEY` in Cell 4 with your own [Roboflow](https://roboflow.com/) API key to download the pre-formatted THEMIS training dataset, or supply your own YOLO-format dataset.
5. Upload `ensemble.py` to the YOLOv5 working directory on Google Drive (or place it at `/content/drive/MyDrive/yolov5/ensemble.py`).

## Workflow

**Step 1 – Dataset generation**

Training image tiles and YOLO-format labels are generated from THEMIS mosaics and the MCD2020 catalogue. Example tiles are provided in `dataset_generation/416_train_val/`.

**Step 2 – Self-training loop**

Run `Auto_iterative_self_training.ipynb` on Google Colab. Each loop:
1. Trains three YOLOv5 models with randomly varied data-augmentation settings (`hsv_v`, `fliplr`).
2. Runs detection on the training images with each model.
3. Calls `ensemble.py` to apply unanimous-consensus filtering (IoU > 0.6 across all three models) and a size-dependent confidence threshold (0.2 for small craters, 0.5 for medium/large).
4. Merges accepted pseudo-labels with the existing ground-truth labels for the next iteration.

The loop repeats until precision and recall plateau (typically ~24 iterations for THEMIS).

**Step 3 – Multi-resolution detection**

Apply the final trained model to large THEMIS mosaic tiles at three resolutions (416×416, 1024×1024, 6144×6144 pixels) to detect craters across the full size range. Follow the instructions in the multi-resolution detection notebook (link to be updated).
